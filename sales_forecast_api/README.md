# Sales Forecasting API

A production-ready Flask REST API for predicting daily retail store sales using a trained LightGBM model.

---

## Project Structure

```
sales_forecast_api/
├── app.py               ← Flask application (API server)
├── train_model.py       ← Model training & serialization script
├── requirements.txt     ← Python dependencies
├── models/              ← Serialized model artifacts (generated by train_model.py)
│   ├── lgb_model.pkl
│   ├── feature_names.pkl
│   ├── feature_names.json
│   ├── store_stats.json
│   └── metrics.json
├── static/
│   ├── css/style.css    ← Dashboard stylesheet
│   └── js/dashboard.js  ← Dashboard interactivity
└── templates/
    └── index.html       ← Interactive web dashboard
```

---

## Quick Start

### 1. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Train and serialize the model

Place `TRAIN.csv` in the project root, then run:

```bash
python train_model.py --data TRAIN.csv
```

This writes all model artifacts to the `models/` directory.

### 3. Start the API server

```bash
python app.py
```

The server starts at `http://localhost:5000`.

### 4. (Production) Run with Gunicorn

```bash
gunicorn app:app -w 4 -b 0.0.0.0:5000
```

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | Interactive dashboard UI |
| GET | `/health` | Service health check |
| GET | `/model/info` | Model metadata & validation metrics |
| POST | `/predict` | Single-record prediction (JSON) |
| POST | `/predict/batch` | Batch predictions (JSON array or CSV upload) |

---

### Single prediction

```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "Store_id": 1,
    "Store_Type": "S4",
    "Location_Type": "L1",
    "Region_Code": "R1",
    "Date": "2019-06-15",
    "Holiday": 0,
    "Discount": "Yes"
  }'
```

**Response:**
```json
{
  "predicted_sales": 67842.50,
  "store_id": 1,
  "date": "2019-06-15",
  "store_type": "S4",
  "location_type": "L1",
  "region_code": "R1",
  "discount": "Yes",
  "holiday": 0,
  "inference_time_ms": 14.2
}
```

---

### Batch prediction — JSON array

```bash
curl -X POST http://localhost:5000/predict/batch \
  -H "Content-Type: application/json" \
  -d '[
    {"Store_id": 1, "Store_Type": "S4", "Location_Type": "L1", "Region_Code": "R1", "Date": "2019-06-01", "Holiday": 0, "Discount": "Yes"},
    {"Store_id": 2, "Store_Type": "S1", "Location_Type": "L2", "Region_Code": "R2", "Date": "2019-06-01", "Holiday": 0, "Discount": "No"}
  ]'
```

**Response:**
```json
{
  "predictions": [
    {"store_id": 1, "date": "2019-06-01", "store_type": "S4", "discount": "Yes", "holiday": 0, "predicted_sales": 67842.50},
    {"store_id": 2, "date": "2019-06-01", "store_type": "S1", "discount": "No",  "holiday": 0, "predicted_sales": 34210.00}
  ],
  "count": 2,
  "inference_time_ms": 18.7
}
```

---

### Batch prediction — CSV upload

```bash
curl -X POST http://localhost:5000/predict/batch \
  -F "file=@TEST_FINAL.csv"
```

The CSV must contain these columns:
`Store_id, Store_Type, Location_Type, Region_Code, Date, Holiday, Discount`

---

## Input Schema

| Field | Type | Required | Values |
|-------|------|----------|--------|
| `Store_id` | int | Yes | 1–365 |
| `Store_Type` | string | Yes | S1, S2, S3, S4 |
| `Location_Type` | string | Yes | L1, L2, L3, L4, L5 |
| `Region_Code` | string | Yes | R1, R2, R3, R4 |
| `Date` | string | Yes | YYYY-MM-DD |
| `Holiday` | int | No (default: 0) | 0 or 1 |
| `Discount` | string | No (default: "No") | "Yes" or "No" |

## Error Handling

All errors return a JSON response with an `error` field and appropriate HTTP status codes:

| Code | Meaning |
|------|---------|
| 200 | Success |
| 400 | Bad request (invalid JSON / missing body) |
| 404 | Endpoint not found |
| 405 | Method not allowed |
| 413 | Batch too large (> 5,000 records) |
| 422 | Validation error (invalid field values) |
| 500 | Internal server error |

## Extending the API

- **Add authentication:** Use `flask-httpauth` or JWT tokens for protected endpoints.
- **Add logging:** Replace `app.logger` with structured logging via `python-json-logger`.
- **Add caching:** Cache model predictions for repeated identical inputs using `flask-caching`.
- **Deploy to cloud:** Package as a Docker container and deploy to AWS ECS, GCP Cloud Run, or Azure App Service.
